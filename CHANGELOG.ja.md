# リリースノート


## 1.14.3

- **修正:** RSSI 中央値フィルタを 3 サンプルから 5 サンプルに増加 — 信号表示がより滑らかに、単発の外れ値を完全に抑制。
- **修正:** ログイン時起動を有効にしてもアプリが応答停止しなくなりました。
- **修正:** デバイスリスト展開時の偶発的なフリーズ（高速開閉時など）を修正。
- **修正:** UUID ローテーション時に古いエントリを適切にクリーンアップし、重複デバイスの蓄積を防止。
- **修正:** 親メニューを閉じた後にデバイスサブメニューが予期せず再オープンする問題を修正。
- **修正:** Option キー詳細表示後にサブメニュー幅が正しくリセットされるように修正。
- **スタイル:** MAC アドレス解決済みデバイスを標準テキスト色、未解決の UUID のみのデバイスをグレーで表示 — ダークモードでも一目で区別可能に。


## 1.14.2

- **修正:** RSSI 信号表示がより安定 — 5 サンプル移動平均から中央値フィルタ（3 サンプル）に変更。単発の異常値を完全に除去し、実際の信号変化への応答遅延も低減。
- **修正:** 監視対象デバイスの Bluetooth UUID がローテーション（プライバシー機能）した際、アプリがリアルタイムで新しい UUID を検出・継承。以前は再起動まで「未検出」と表示されることがありました。
- **修正:** 画面ロック中もバックグラウンドスキャンが継続 — 監視デバイスが設定されている限り、UUID ローテーションと近接検出が動作し続けます。
- **修正:** RSSI 0 dBm を表示しないよう修正（切断として扱います）。
- **修正:** 区切り線と「スキャン中…」がすべてのケースで正しく表示 — 未チェックデバイス到着時、メニュートラッキング中に初回デバイスをチェックした時、チェック済みデバイスがない時。
- **修正:** 同じ名前でクラシック Bluetooth と BLE の両方をブロードキャストするデバイス（例: RedMagic）の MAC 解決に失敗する問題 — クラシックエントリを優先。
- **修正:** メニューオープン時のデバイス検出カウント（例: 「1/2」）が即座に正確に。
- **修正:** スキャン中の監視デバイスリストの並び順がちらつかなくなりました。

- **新機能:** ⌥ (Option) を押すと完全な MAC アドレスと UUID を表示 — デフォルトではデバイス名と RSSI のみ。
- **新機能:** UUID のみのデバイスは折りたたみ形式 `XXXX…YYYY` で表示。
- **新機能:** クリック可能な「ペアリングでランダムアドレスを安定追跡 →」ヒントで Bluetooth 設定を開く。
- **新機能:** メインメニュー「デバイスリスト」に ⌥ インジケーターを追加。


## 1.14.1

- IOBluetooth 経由でシステムのペアリング済み Bluetooth デバイスから MAC アドレスを解決し、デバイスリストに表示。
- 切断や再起動後に BLE UUID が変更された場合、MAC ベースの相互相関で自動的にデバイス追跡を再マッピング。再設定不要。
- 監視デバイスをデバイスリストの上部にソートし、未監視デバイスは発見順に下に表示。
- Bluetooth エンタイトルメントを追加し、より広範な macOS バージョンとの互換性を確保。
- MAC 永続化を修正: `{UUID → MAC}` から `{MAC → UUID}` 形式に変更し、書き込み時にマージ。同時 BLE 関連付け時に MAC エントリが上書きされるのを防止。
- 監視デバイスが常にランタイムデバイス辞書に含まれるようにし、再起動後に MAC 相互相関がローテーション済み BLE UUID をマッチできるように。
- 高速再オープン時の未監視デバイス非更新問題を修正。期限切れデバイスのクリーンアップをメニューが閉じている時に延期。
- 初回起動時にチェック済み/未チェックデバイス間の区切り線が表示されない問題を修正。
- メニュートラッキング中の安定性を改善: メニューオープン中は区切り線の表示のみ更新し、完全な再構築は延期。
- IOBluetooth 名前検索失敗時に macOS Bluetooth LE データベースにフォールバックして MAC アドレスを解決。

## 1.13.6

- 「アップデート」サブメニューを追加。自動更新チェックと手動更新アクションを統合。

- 自動更新通知をサイレントにし、中断を軽減。
- BLE 名前解決ログを現在のユーザーの Library/Logs ディレクトリに移動。

- iBeacon プレフィックス解析の修正とランチャーパス検証の改善。
- アプリバンドル ID を `com.github.Skyearn.BLEUnlock` に移行し、従来の設定、キーチェーンデータ、ログイン項目との互換性を確保。

## 1.13.1

- スリープ/ウェイク境界での積極的なディスプレイウェイクリトライを抑制し、ディスプレイが「起きたが実際には点灯していない」半覚醒状態になるのを防止。

- スキャンキャッシュエントリの期限切れ時にメニューバーサマリーと監視デバイスリストが一時的に不一致になる問題を修正。
- 検出カウントと最強 RSSI をより明確に表示するよう監視デバイスサマリーテキストを改善。


## 1.13.0

- 複数の BLE デバイスを同時に監視可能にし、ロック解除・ロックロジックを設定可能に。
- スリープ復帰時のロック解除信頼性を改善 — ロック画面の準備が整うのを待ってからパスワードを送信し、初回が早すぎた場合はリトライ。
- メニューバーサマリーを、選択デバイス数、検出デバイス数、最強監視信号を示す 1 行のステータスに簡素化。
- サマリー行とデバイスリスト間の RSSI 表示を統一し、値の理解を容易に。
- 監視中だが現在未検出のデバイスを表示しつつ、デバイススキャンリストのライブ更新を復元。
- 簡体字中国語 README を追加し、英語ドキュメントからリンク。
